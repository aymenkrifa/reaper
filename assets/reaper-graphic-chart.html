1the mark

concept

The mark is a skull drawn on a grid of terminal character cells — reaper's entire interface rendered down to one glyph. Its right eye is a 2 × 2 phosphor-green block cursor: the terminal is alive inside the reaper, waiting for input. Everything in the identity flows from this one idea — the tool is the terminal, so the brand is built from the terminal's own material: cells, monospace, phosphor.

construction

The skull occupies an 11 × 10 cell grid. Cells are square with a gap of ⅛ pitch, which keeps the character-grid texture visible at any size. Minimum clear space around the mark is 2 cells on every side. Never redraw the pixel map — always scale the master asset.

2logo & lockups

primary lockup

Mark + wordmark, horizontally locked. The wordmark is JetBrains Mono Bold, always lowercase, converted to outlines so it renders identically everywhere — including inside GitHub READMEs, where SVGs cannot load fonts.

on dark — default. bone #EAE4D6 on any background darker than #333.
on light — ink #141815, eye deepens to #149E52 for contrast.

icon

For avatars, favicons and social tiles: the mark centered on a phosphor-black rounded tile. Use the bare mark (transparent) when the surface already provides a dark background.

reaper-icon.svg — self-contained tile, works on any surface.
reaper-mark.svg — transparent, for dark surfaces you control.

readme banner

The banner frames the lockup inside a TUI panel border, with reaper's real keybindings as a status bar. Drop it at the top of the README at full width.

minimum sizes

Mark ≥ 16 px tall · lockup ≥ 24 px tall · banner ≥ 640 px wide. Below these, the cell gaps close up and the cursor eye stops reading.

light mode

Print and light surfaces swap to ink #141815 on paper #F2EFE6, with green #149E52 and red #DE3B36. Dark is always the default; light is the exception.

4typography

One family carries the brand: JetBrains Mono — an open-source face designed for terminals, which is exactly where reaper lives. It is paired with IBM Plex Sans for long-form reading only (docs, articles), never for the interface or the brand voice.

display · JetBrains Mono Bold 700–800 · always lowercase · tracking −2%
kill the port, keep the peace
interface & code · JetBrains Mono Regular 400–500
PID 48213 · node · LISTEN :3000 → SIGKILL sent // press [r] to refresh
long-form body · IBM Plex Sans 400
Plex Sans steps in only when a paragraph runs longer than a terminal ever should — release notes, guides, this document. It shares Plex Mono's skeleton, so it sits comfortably next to code without imitating it.
5voice & tui conventions

voice

Lowercase, imperative, terse. reaper names itself in lowercase everywhere — even at the start of a sentence. Dark humor is welcome in small doses ("3 processes reaped"), drama is not. States are UPPERCASE (LISTEN), signals keep their POSIX names (SIGKILL), key hints live in brackets: [q] quit.

panel grammar

Any branded surface — website section, slide, social card — is drawn as a TUI panel: a thin ash border, a title notched into the top edge, key hints along the bottom. This document follows the same grammar.

6rules & assets

do / don't

use master assets, keep the gaps
don't rotate or tilt
don't recolor the palette
don't add glows or shadows
don't stretch or condense
don't place on busy surfaces

Two more that matter: the cursor eye is always green — it may flip to signal red only in an animated "kill" moment, never at rest. And the wordmark is never set in another typeface, never capitalized, never "Reaper".
